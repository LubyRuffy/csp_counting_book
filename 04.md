# 第四课 为什么长方形不用一个个画？

## 第一部分 国王想把农田租出去

新仓库验收后的第二天，建筑师带着托马斯去了王国南边。

过去，那里的农田长得十分自由。有一块像长方形，有一块像楼梯，还有一块为了绕开池塘、树桩和废弃矿洞，最后变成了某种很难用语言描述的形状。那块地的主人坚持说它像一只正在奔跑的羊，财政大臣则认为它更像一张无法计算税额的灾难。

农夫们平时依靠树木、水沟和记忆划分土地。东边那棵橡树属于谁，池塘旁边三步远的地方是谁家的，大家似乎都很清楚。直到有一天，一名木匠砍掉了那棵橡树，三户村民同时宣布自己多出了一块地。

国王终于决定重新规划农田。

建筑师让工人铺设石砖小路，把整片土地整整齐齐地分成四行六列。每个小方格大小相同，方格之间的小路既是边界，也能防止村民为了少走两步，直接踩过别人刚种好的小麦。

托马斯来到农田时，二十四个小方格已经全部整理完毕。几名村民抱着种子站在田边，一名农夫牵着牛，另一名农夫则在努力把一只猪从胡萝卜地里劝出来。

猪显然认为，既然胡萝卜已经长在这里，就不应该再进行复杂的所有权讨论。

国王站在新农田中央，手里拿着土地登记册。

“有些农夫只需要一小块地，有些需要两行三列，还有些想承包更大的区域。”他说，“所有出租土地都必须是长方形。我想知道，这片农田一共能划出多少块不同的长方形土地。”

托马斯看了看四行六列的方格。

“二十四块。”

建筑师没有反驳，只走到第一行，在相邻的两个方格外插上四根木桩。

“这一整块是不是长方形？”

“是。”

建筑师又把木桩向外移动，围出两行三列的区域。

“这块呢？”

“也是。”

他继续扩大边界，最后把整片四行六列的农田全部围了起来。

“这个呢？”

托马斯看着最外侧的石砖路。“当然也是。”

国王合上登记册。“所以二十四只是最小的那些。”

托马斯这才意识到，自己数的是最小方格的数量，而国王问的是：这些小方格能够组成多少个不同位置、不同大小的长方形。

四乘六确实等于二十四。

计算没有错。

只是回答了另一个问题。

一头牛恰好在这时跨过围栏，落在刚刚翻好的耕地上。湿润的农田立刻被踩回普通泥土。

建筑师闭上眼睛，深深吸了一口气。

托马斯问：“这会影响长方形数量吗？”

建筑师睁开眼睛，看着那头正在若无其事嚼草的牛。“不会影响形状，只会影响我的心情。”

托马斯拿出笔记本，决定先从最小的长方形开始统计。

一行一列的有二十四个。

一行两列的，第一行可以从最左边开始，也可以向右移动一格、两格，一直移动到最右边。其他三行也同样如此。

接下来还有一行三列、一行四列、一行五列和一行六列。

数完一行高的，还要继续统计两行高、三行高和四行高的情况。

托马斯画了一张表，准备按照“高度”和“宽度”分类。这个办法理论上可行，但数字越来越多。他必须同时记住某种大小的长方形能上下移动多少次、左右移动多少次，还要防止把两行三列和三行两列混在一起。

他写了一会儿，决定换一种更直观的方法。

“我把所有长方形画出来。”

建筑师递给他一摞农田图纸。托马斯先用红色圈出所有一行一列的长方形，再用蓝色圈一行两列，绿色圈一行三列。没过多久，红线、蓝线和绿线便互相穿过，整张图纸像是一只洞穴蜘蛛在喝下迅捷药水后完成的作品。

他又向牧羊人借来了更多染料。

牧羊人提醒道：“羊毛只有十六种常用颜色。”

“长方形显然不止十六个。”

“那就重复使用颜色。”

“重复以后我怎么知道自己有没有画过？”

牧羊人抱着染料思考了一会儿，发出一声很像村民的“哼”。长时间和村民交易，显然会对人的语言系统产生影响。

托马斯只好改成一张图纸只画一个长方形。

这样确实不会相互重叠，但很快，田边就铺满了纸。风从平原吹过，几十张图纸同时飞了起来。有的掉进水渠，有的贴在铁傀儡脸上，还有一张被那头越过围栏的牛嚼掉了一个角。

铁傀儡缓慢地把图纸从脸上取下来，看了一眼，再递还给托马斯。纸上原本画着一个两行四列的长方形，现在只剩下三条半边界。

托马斯望着满地飞舞的图纸，终于承认：把每一个长方形画出来虽然不是做不到，却很难保证不重不漏。两个长方形可能大小相同，但位置不同；也可能位置接近，只差一格。图画得越多，检查反而越困难。

他需要的不是更多颜色，也不是更多纸。

他需要找到一小组信息，能够唯一决定一个长方形。

Notch正坐在水渠边，手里拿着一根小麦。他似乎已经在那里看了很久，只是一直没有打扰托马斯与图纸、风和牛之间的斗争。

他指向一块两行三列的土地。

“如果把里面的小麦全部换成胡萝卜，它还是原来的长方形吗？”

“是。”

“如果牛踩坏其中一个方格呢？”

托马斯看了一眼那块变回泥土的耕地。“形状仍然没变。”

Notch起身走到田边，把右侧的一根木桩向左移动了一格。

“现在呢？”

“两行三列变成了两行两列，是另一个长方形。”

他又把上方木桩向下移动一格。

“现在只剩一行两列，又变了。”

Notch把木桩放回原处，没有解释公式，也没有告诉托马斯应该怎样计算。

他只问道：“你画了那么多内部方格，可真正让长方形发生变化的，是什么？”

托马斯低头看向纵横交错的石砖小路。

长方形内部种小麦还是胡萝卜，并不重要。

某个方格暂时变成泥土，也不改变它的边界。

真正决定一块长方形土地的，是围住它的四条线：

上边界。

下边界。

左边界。

右边界。

只要这四条边界确定，中间的区域就完全确定了。

Notch把小麦交给旁边的村民，转身向王城走去。

“别再数里面的格子了。”

托马斯问：“那数什么？”

Notch没有回头。

“数那些真正决定它的东西。”

## 第二部分 四条边界决定一个长方形

托马斯重新站到农田边缘。

刚才，他一直看见四行六列、二十四个方格。现在，他第一次认真注意那些分隔方格的线。

四行方格并不只有四条横线。

最上面有一条外边界，四行方格之间有三条分界线，最下面还有一条外边界。

因此，四行方格一共有五条横向边界。

同样，六列方格共有七条竖向边界。

托马斯给横线从上到下编号一到五，给竖线从左到右编号一到七。

要确定一个长方形的高度，只需要选择两条不同的横线。较上面的一条自然成为上边界，较下面的一条成为下边界。

五条横线中，第一条可以和下面四条中的任何一条配对，共四种。

第二条不必再和第一条配对，因为那一对已经数过了。它还可以和后面的三条配对。

第三条有两种。

第四条只剩一种。

第五条下面没有其他横线。

所以，上下边界共有：

\[
4+3+2+1=10
\]

种选法。

竖线也是同样的道理。

七条竖线中，第一条可以和后面六条配对，第二条还有五种，接下来是四种、三种、两种和一种。

因此，左右边界共有：

\[
6+5+4+3+2+1=21
\]

种选法。

每一种上下边界，都可以搭配所有二十一种左右边界。只要两条横线和两条竖线确定，就会围出唯一的长方形。

因此，长方形总数是：

\[
10\times21=210
\]

托马斯盯着这个答案看了很久。

他没有画出二百一十个长方形，也没有逐个把它们列在账本上。他只是数清了横向边界有多少种选择，又数清了竖向边界有多少种选择。

二百一十个复杂对象，被转换成了两次简单的“选两条线”。

建筑师拿来托马斯之前按大小分类的草稿。他们把剩余部分补完，再将所有尺寸的数量相加，结果同样是二百一十。

两种不同的数法，得到了同一个答案。

第一种方法先决定长方形有几行、几列，再计算这种大小能够放在多少个位置。

第二种方法不直接关心长方形的大小，只选择决定它的四条边界。

两种方法都正确，但数边界更接近长方形真正的身份。

因为每一个长方形，都有且只有一组上、下、左、右边界。

反过来，每一组两横两竖，也只会产生一个长方形。

不会有两个不同的长方形共享完全相同的四条边界，也不会有哪个长方形找不到自己的四条边界。

这又是一种“不重、不漏”。

第一课中，每件木材必须恰好进入一个类别。

现在，每个长方形必须恰好对应一组边界。

当两类对象能够这样一一对应时，数清其中一种，就等于数清另一种。

托马斯在笔记本上写下：

**不要总盯着难数的对象。**

**寻找能够唯一决定它的另一种东西。**

国王走进田间，站在一块三行四列的区域里。

“这块地里面有十二个小方格。”他说，“为什么不能用十二来代表它？”

“因为面积不能唯一决定长方形。”托马斯回答，“一行十二列也是十二个方格，两行六列也是十二个方格，它们的形状不同。就算形状一样，位置也可能不同。”

“那什么能够唯一决定？”

“四条边界。”

国王点了点头，又问：“如果农田不是四行六列，而是 \(n\) 行、\(m\) 列呢？”

托马斯沿着刚才的思路继续推广。

\(n\) 行方格共有 \(n+1\) 条横线。

第一条横线可以和后面 \(n\) 条配对，第二条有 \(n-1\) 种，接着不断减少，直到一。

因此，上下边界的选择总数是：

\[
n+(n-1)+\cdots+2+1
\]

也就是：

\[
\frac{n(n+1)}2
\]

同样，\(m\) 列方格的左右边界共有：

\[
\frac{m(m+1)}2
\]

种选择。

所以，一个 \(n\) 行、\(m\) 列的方格区域中，所有长方形的数量是：

\[
\frac{n(n+1)}2
\times
\frac{m(m+1)}2
\]

托马斯特意在公式旁边写下一句话：

**公式不是思考的起点。**

**它只是“选两条横线，再选两条竖线”的缩写。**

如果只记得公式，却忘了边界，题目稍微换一种外表，公式便会立刻变成一串没有方向的符号。

建筑师盯着农田看了一会儿，忽然问道：“这些长方形里，有多少是正方形？”

托马斯没有立即回答。

正方形当然也属于长方形。刚才选择边界时，它们已经包含在二百一十个结果里面。

但普通长方形只要求选择两横两竖，正方形还多出一条条件：

上下边界之间的距离，必须等于左右边界之间的距离。

托马斯决定按照边长分类。

边长为一格的正方形，就是二十四个最小方格：

\[
4\times6=24
\]

边长为两格的正方形，占两行两列。它的上边可以放在前三行，左边可以放在前五列，所以有：

\[
3\times5=15
\]

个。

边长为三格时，纵向有两个位置，横向有四个位置：

\[
2\times4=8
\]

个。

边长为四格时，纵向只剩一个位置，横向有三个位置：

\[
1\times3=3
\]

个。

农田只有四行，不可能容纳边长五格的正方形。即便那头牛继续踩坏农田，也不会让土地凭空多长出一行。

所以，正方形共有：

\[
24+15+8+3=50
\]

个。

国王问：“那不是正方形的长方形呢？”

托马斯原本想重新逐类统计长和宽不同的情况，写下第一行后便停住了。

所有长方形共有二百一十个。

其中五十个是正方形。

剩下的自然是：

\[
210-50=160
\]

个非正方形长方形。

这一次，他没有正面统计目标，而是先数更容易得到的全部，再去掉不需要的特殊部分。

托马斯隐约觉得，这种“先数全部，再减去一部分”的方法，将来还会遇到。数学似乎很喜欢绕路，尤其当正面道路上挤满了分类、遗漏和令人烦躁的例外时。

财政大臣看完报告后，最先注意到的不是二百一十，也不是五十。

他问：“这些二百一十种土地能同时出租吗？”

建筑师摇头。“不能。它们会相互重叠。”

财政大臣显得有些失望。

计数可以告诉王国有多少种选择，却不会因为财政大臣希望增加收入，就让二十四块小方格同时变成二百一十份租金。数学偶尔会制造巨大的数字，现实则负责阻止人们把它们全部写进预算。

## 第三部分 程序员时间：让机器移动四条边界

下午，托马斯把农田图纸带到了皇家红石实验室。

红石工程师已经造好了一台自动土地划分机。机器由四条能够移动的栅栏组成，两条横着，两条竖着。每拉动一次控制杆，活塞便会移动栅栏，围出一个新的长方形。

“它能依次展示全部二百一十块土地。”工程师说道。

“需要多久？”

“如果栅栏不被活塞夹住，大约半小时。”

工程师拉下控制杆。

第一次，机器成功围出一个一行一列的小方格。

第二次，右侧栅栏向外移动，围出一行两列。

第三次，左侧活塞比右侧慢了一拍，两条栅栏斜着卡进石砖路。第四次，机器把一名站得太近的村民围在中央。

村民抱着一袋种子，看了看四周突然出现的栅栏。

“哼。”

红石工程师立即关闭装置。

“主要结构没有问题。”

托马斯把村民从栅栏里放出来。“任何需要先解救村民的机器，都不应该这么快通过验收。”

他们决定先用程序模拟四条边界的选择：

```cpp
#include <iostream>
using namespace std;

int main() {
    int rows = 4;
    int cols = 6;
    int rectangles = 0;
    int squares = 0;

    for (int top = 0; top < rows; top++) {
        for (int bottom = top + 1;
             bottom <= rows; bottom++) {
            for (int left = 0; left < cols; left++) {
                for (int right = left + 1;
                     right <= cols; right++) {
                    rectangles++;

                    if (bottom - top == right - left) {
                        squares++;
                    }
                }
            }
        }
    }

    cout << rectangles << '\n';
    cout << squares << '\n';
}
```

程序输出：

```text
210
50
```

程序中的四层循环分别选择上边界、下边界、左边界和右边界。每当四条边界全部确定，一个长方形便被统计一次。

如果上下边界之间的距离等于左右边界之间的距离，这个长方形同时还是正方形。

托马斯观察了一会儿。

“程序真的把所有边界组合都走了一遍。”

工程师点点头。

“公式没有逐个走，却直接算出了边界组合的数量。”

“一个适合快速得到总数，一个适合列出或检查每一种。”

他们又把农田改成一行一列。

程序输出一个长方形，也是一个正方形。

改成一行两列时，程序输出三个长方形：左边一个小方格、右边一个小方格，以及整个一行两列的大长方形。其中两个是正方形。

这些小规模结果可以直接用眼睛检查，与程序完全一致。

Notch站在实验室门口，拿起一张画满方格的图纸。

“程序数的真是长方形吗？”他问。

托马斯想了想。

“严格来说，它数的是四条合法边界。”

“为什么这就等于长方形数量？”

“因为每一组合法边界只对应一个长方形，每个长方形也只有一组这样的边界。”

Notch点了点头。

“机器枚举的是容易描述的东西。”

“人真正需要的是与它一一对应的东西。”

这一次，托马斯没有再把二百一十看成一个神奇答案。它只是边界选择数量的结果。

当一个对象难以直接统计时，不妨寻找另一种更简单、又能唯一决定它的对象。

有时，真正值得数的，不是你眼前看到的东西。

而是决定它存在的那些条件。

当天傍晚，国王又派人送来了一份新任务。

王国准备举行建城纪念仪式。六名骑士要站在城堡前方迎接使者，其中一人站在最前面举王国旗帜，另一人站在第二位吹号角，其余人依次排列。

托马斯看着六名骑士的名字。

如果只是选择哪六个人参加，站位似乎并不重要。

可现在，第一个位置代表旗手，第二个位置代表号手，后面的站位也决定了仪仗队的顺序。

两名骑士只要交换位置，结果便可能完全不同。

Notch从门外经过，只看了一眼任务单。

“边界的位置能够决定长方形。”

他指了指名单中的第一个位置。

“那么，一个人在队伍中的位置，会不会也决定答案？”

托马斯望着那六个尚未填写的空位。

上一章中，整齐让大量对象可以一起统计。

这一章中，边界决定了一个图形。

而下一次，他必须面对另一种东西：

**同样一群人，只要站的位置不同，为什么就会变成不同的结果？**
