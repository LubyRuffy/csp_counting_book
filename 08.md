# 第八课 为什么连续的一段只需要两个边界？

## 第一部分 一条铁路究竟需要多少种车票？

远征背包整理完毕后的第五天，王国第一条长途铁路正式开通。

铁路从王城出发，一路穿过平原村、橡木林、白桦镇、沼泽口、沙漠村和峡谷，经过雪原前哨站，最后抵达北方矿山。全线共有九座车站，八段铁轨。

为了这条铁路，红石工程师忙了整整一个月。他铺设普通铁轨、充能铁轨和探测铁轨，安装自动发车装置，还在每座车站修建了颜色不同的站牌。王城站使用红色羊毛，平原村站使用绿色羊毛，雪原站则使用白色羊毛。北方矿山原本计划使用黑色羊毛，后来因为站台太暗，三名乘客连续撞上同一根柱子，只好改成了明亮的黄色。

开通仪式当天，国王、建筑师、财政大臣和几十名村民挤在王城站台上。红石工程师站在一辆崭新的矿车旁，向众人介绍自动发车系统。

“乘客只需坐进矿车，按下按钮，充能铁轨就会把矿车平稳送出车站。”

国王问：“刹车装置测试过吗？”

“测试过。”

“成功了吗？”

“矿车确实停下来了。”

“停在哪里？”

红石工程师指向站台外刚补种好的南瓜地。

国王看了看矿车，又看了看南瓜地，决定先把“高速试运行”改成“低速观摩”。在工程技术尚未完全成熟时，降低速度通常是一种非常成熟的管理技术。

第一位试乘者是一名抱着绿宝石的村民。他刚坐进矿车，一只鸡便从站台下面钻出来，跳到他旁边的空位上。村民转头看着鸡，发出一声不太满意的“哼”。

鸡没有下车。

它显然认为，只要座位空着，自己就已经拥有了乘车资格。对于不会购买车票的生物而言，这是一种十分便利的交通理论。

红石工程师按下按钮。矿车向前滑出两格，忽然停住，接着缓慢地倒回站台。

“为什么又回来了？”托马斯问。

工程师蹲下检查线路。“有一段充能铁轨没有获得红石信号。”

“刚才不是说已经测试过了吗？”

“测试的是刹车。”

这一点倒没有说错。矿车确实停得非常彻底，甚至没有真正离开王城站。村民仍然坐在车里，表情比出发前严肃了一些；那只鸡却显得很满意，毕竟它已经免费完成了一次往返旅行，而且没有离开站台。

铁路重新调试以后，总算能够正常运行。国王随即把托马斯叫到售票处。

售票窗口后面摆着一叠空白车票，每张票上都有两个位置：

**出发站：________**

**到达站：________**

“售票员需要提前准备所有种类的单程车票。”国王说道，“这条铁路一共有九座车站，目前只能从编号较小的车站前往编号较大的车站。一共需要多少种不同车票？”

托马斯看向墙上的线路图。

九座车站之间共有八段铁轨。

他想了想，说道：“八种。”

售票员从窗口后面探出头来。“为什么是八种？”

“因为铁路有八段。每段铁轨对应一段旅程。”

售票员拿起一张空白票。“那从王城到白桦镇呢？”

托马斯沿着线路图看过去。王城到白桦镇要经过平原村和橡木林，共跨越三段铁轨。

“这当然也应该有一张票。”

“王城到北方矿山呢？”

“也应该有。”

“沼泽口到峡谷？”

“也算。”

托马斯意识到，自己又数错了对象。

八段铁轨只是整条铁路最小的组成部分。一张车票可以只跨越一段，也可以连续经过两段、三段，甚至从第一站一直坐到最后一站。

国王要数的不是铁轨有多少段，而是旅客能够选择多少种不同的出发站和到达站。

托马斯拿出笔记本，决定从王城站开始列举。

王城到平原村。

王城到橡木林。

王城到白桦镇。

王城到沼泽口。

他一直写到王城到北方矿山，共八张车票。

接下来从平原村出发：

平原村到橡木林。

平原村到白桦镇。

平原村到沼泽口。

这样继续下去，似乎总能把全部车票写完。

可是售票窗口外的队伍也在不断变长。

一名村民敲了敲柜台。

“哼。”

售票员问：“去哪一站？”

村民先指了指王城站牌，又指了指自己脚下。

“您现在就在王城。”

村民点点头。

“您想买王城到王城的车票？”

村民再次点头，把一颗绿宝石放到柜台上。

售票员沉默了一会儿，显然正在认真权衡铁路规定和一颗绿宝石之间的关系。

托马斯赶紧解释：“起点和终点不能是同一座车站。否则旅客根本没有移动，也就没有发生旅行。”

村民收回绿宝石，发出一声充满遗憾的“哼”。他大概只是想花钱购买一种不必离开王城的旅行体验。考虑到矿车目前的可靠性，这种要求并非完全不能理解。

托马斯继续列票。他很快发现，自己必须格外小心。

王城到雪原前哨站算一种。

雪原前哨站到王城却不能再算，因为铁路目前只向编号增大的方向行驶。返程线路还没有完成。

“为什么返程还没开通？”托马斯问。

售票员看向红石工程师。

工程师正在远处检查一块完全不需要检查的站牌。

“据说返程启动按钮被安装在北方矿山。”售票员说道。

“那矿山的人怎么回来？”

“他们可以按按钮。”

“矿车回来以后，下一个人怎么去矿山按按钮？”

售票员没有回答。

有些红石装置在第一次使用时十分完美，第二次使用则要求另一名红石工程师提前站在终点。这并不是错误，只是一种对工作人员数量要求较高的自动化方案。

托马斯重新看向自己写出的长名单。

一张张列当然能够得到答案，但随着车站增加，检查会越来越麻烦。他可能漏掉某一对车站，也可能把相同方向的车票写两遍。

线路图上只有九座车站，真正决定一张车票的，似乎只是其中两座。

一座是出发站。

一座是到达站。

而由于铁路只向前行驶，只要选定两座不同车站，编号较小的自然成为出发站，编号较大的自然成为到达站。

中间经过哪些站，不需要额外选择。

铁路只有一条。

出发站和到达站确定以后，中间那一整段连续旅程也就跟着确定了。

这一次，Notch并没有出现。

托马斯自己把笔记本翻到前几章，找到第四课中数长方形时写下的那句话：

**不要总盯着难数的对象。**

**寻找能够唯一决定它的另一种东西。**

一个长方形由四条边界决定。

一张单程车票，则由两座车站决定。

他放下那张越来越长的车票清单，开始数车站对。

## 第二部分 两个端点为什么能决定中间的一切？

九座车站中，从王城出发，可以选择后面的八座车站作为终点，因此有八种车票。

从第二站平原村出发，前面的王城不能再作为终点，只能选择后面的七座车站。

从第三站出发，有六种。

随后依次是五种、四种、三种、两种和一种。

最后一座北方矿山后面已经没有车站，所以不能继续产生新的向前车票。

所有车票共有：

\[
8+7+6+5+4+3+2+1=36
\]

种。

这个过程也可以使用上一章学过的组合来理解。

从九座车站中选择两座：

\[
\binom{9}{2}=36
\]

选择以后，不必再为它们安排顺序。位置较靠前的车站自动成为起点，靠后的自动成为终点。

如果选择的是王城和雪原前哨站，那么旅程只能是王城到雪原前哨站。中间会依次经过平原村、橡木林、白桦镇、沼泽口、沙漠村和峡谷，不可能跳过其中某一段铁轨，然后仍然声称这是同一条连续铁路。

托马斯把两根木桩分别放在线路图的两个车站上。

左边木桩确定开始。

右边木桩确定结束。

两根木桩之间的一切都被保留下来。

这就是“连续”的意义。

一段连续旅程不是任意挑选几个车站。它不能选择王城、白桦镇和北方矿山，却跳过它们之间的铁路。那样得到的不是一段旅程，而是一份旅客希望矿车能够瞬移的愿望。

国王看完计算，问道：“如果以后允许双向行驶呢？”

那时，王城到白桦镇与白桦镇到王城将成为两张不同车票。起点和终点的职责不能交换，顺序重新变得重要。

九座车站中，每一座都可以作为起点，再从另外八座中选择终点：

\[
9\times8=72
\]

种。

同样选择两座车站：

不区分方向时，是三十六种。

区分起点和终点时，是七十二种。

数字翻倍，不是因为铁路突然变长，而是因为“从哪里出发”开始具有独立意义。

托马斯在账本旁边写下：

**两个对象是否可以交换，仍然决定着顺序是否重要。**

售票问题解决后，铁路通信员又带来了一张纸条。

“我们从掠夺者前哨站附近截获了一段信号。”他说，“只有七个字母。”

纸条上写着：

```text
CREEPER
```

国王看了一眼。“这是苦力怕。”

“是的。”通信员说道，“但信号可能只是更长消息中的一部分。我们需要检查这串文字中所有可能的连续片段，看看里面是否隐藏着其他暗号。”

窗边的一只鹦鹉听见单词，立刻模仿出一声逼真的：

“嘶——”

房间里的所有人同时后退了一步。

鹦鹉歪着脑袋看着他们，显得非常满意。它准确复制了苦力怕的声音，却完全不理解为什么这段声音会让人类短暂地回忆自己一生中所有没有存档的时刻。

托马斯把七个字母依次写开：

```text
C R E E P E R
```

连续片段可以是：

```text
C
CR
CRE
REE
EEPER
CREEPER
```

但是不能从第一个字母 `C` 跳到第三个字母 `E`，再跳到最后一个 `R`，拼成 `CER`。因为中间的字符被跳过了，它已经不再是原文字中的连续一段。

铁路中的连续旅程由两座车站决定。

文字中的连续片段又由什么决定？

托马斯在每两个字母之间画了一条线，并在整串文字的左右两端也各画一条：

```text
| C | R | E | E | P | E | R |
```

七个字母，一共有八条边界。

选择左边界，再选择它右侧的一条边界，中间的字符便组成一个非空连续片段。

选择第一条和第二条边界，得到：

```text
C
```

选择第一条和第四条边界，得到：

```text
CRE
```

选择第三条和最后一条边界，得到：

```text
EEPER
```

每一对不同边界都会产生一个连续片段。

每一个连续片段也都有唯一的左右边界。

所以，七个字符的所有非空连续片段共有：

\[
\binom{8}{2}=28
\]

个。

为什么铁路有九座车站时选择两座车站，而七个字符却有八条边界？

因为一段铁路旅程的端点正好是车站。九座车站之间的八段铁轨，每一段旅程都从某座车站开始，在后面某座车站结束。

文字中的对象则是字符。一段文字可能只包含第一个字符，因此它的左边界必须位于字符以前；也可能包含最后一个字符，因此右边界必须位于字符以后。

一排 \(n\) 个对象，共有 \(n+1\) 条分界线。

只要选择其中两条不同边界，就能确定一个非空连续片段：

\[
\binom{n+1}{2}
=
\frac{n(n+1)}2
\]

托马斯又从另一个角度验证了这个结果。

长度为一的连续片段有 \(n\) 个。

长度为二的有 \(n-1\) 个。

长度为三的有 \(n-2\) 个。

一直到长度为 \(n\) 的完整片段，只有一个。

因此总数是：

\[
n+(n-1)+(n-2)+\cdots+1
=
\frac{n(n+1)}2
\]

一种方法按照片段长度分类。

另一种方法选择左右边界。

两种方法数的是同一批对象，所以答案相同。

通信员指着单词中的三个字母 `E`。

“单独一个 `E` 有几个？”

托马斯回答：“按位置统计，有三个。第三个字符、第四个字符和第六个字符，分别拥有不同的左右边界。”

“可它们写出来都一样。”

“那要看问题怎样定义‘相同’。”

如果按出现位置统计，三个 `E` 是三个连续片段，因为它们来自三个不同位置。

如果只问有多少种不同的文字内容，那么三个单独的 `E` 只能算一种。

这已经是两个不同的问题。

连续边界只能保证按位置不重不漏，却不会自动替人完成内容去重。数学从不负责猜测题目中的“相同”究竟是什么意思。题目如果说不清楚，人们通常会围绕几个完全一样的字母争论很久，然后怪数学过于复杂。

Notch在傍晚时来到通信站。

他没有询问托马斯答案是多少，而是拿起写着 `CREEPER` 的纸条。

“铁路和文字，有什么相同的地方？”

“它们都有固定顺序。”

“还有呢？”

“从中取连续的一段时，一旦确定开始和结束，中间的所有对象都会自动保留。”

Notch点点头。

“所以你真正数的是什么？”

“端点，或者边界。”

“不是中间那些对象？”

“不需要逐个决定。两个边界已经把它们全部决定了。”

这一次，托马斯发现自己几乎没有等待Notch的提醒。

他已经开始主动寻找：

什么信息能够唯一决定目标？

Notch没有说“很好”，只是把纸条放回桌上，轻轻点了一下头。

这已经足够了。

## 第三部分 程序员时间：让机器移动左右边界

红石工程师听说七个字符共有二十八个连续片段，立即制作了一台自动文字切割机。

机器左侧安装一条长纸带，上面写着：

```text
CREEPER
```

纸带上方有两只可以移动的红石指针。一只表示左边界，另一只表示右边界。每次按下按钮，机器就把两个指针之间的文字复制到另一张纸上。

第一次测试时，两只指针停在同一条边界上。

机器打印出一张空白纸条。

工程师拿起来仔细看了看。

“这是空片段。”

通信员问：“它有什么意义？”

“目前没有。”

“那为什么打印出来？”

“因为两条边界允许重合。”

托马斯摇头。“我们这次统计的是非空连续片段。右边界必须严格位于左边界右侧。”

工程师调整装置。

第二次，机器顺利打印：

```text
C
CR
CRE
CREE
```

接着，纸带末端忽然多出一只鸡。机器打印出：

```text
CREEPER鸡
```

红石工程师看向传送带。那只鸡正站在纸带末端，仿佛它本来就是信号中的最后一个字符。

“外部输入干扰。”工程师说道。

托马斯把鸡抱下来。“模型里仍然没有鸡。”

工程师已经开始习惯这句话了。

他们用一段短程序完成相同的枚举：

```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    string text;
    cin >> text;

    int total = 0;

    for (int left = 0; left < text.size(); left++) {
        for (int right = left;
             right < text.size(); right++) {
            cout << text.substr(
                left, right - left + 1
            ) << '\n';

            total++;
        }
    }

    cout << "共 " << total
         << " 个连续片段\n";
}
```

输入：

```text
CREEPER
```

程序首先固定左边界在第一个字符以前，让右边界不断向右移动，因此依次输出：

```text
C
CR
CRE
CREE
CREEP
CREEPE
CREEPER
```

然后左边界向右移动一个位置，再输出所有以 `R` 开始的连续片段。

直到最后一个字符单独成为一段。

程序最终得到：

```text
共 28 个连续片段
```

托马斯观察两层循环。

外层决定从哪里开始。

内层决定在哪里结束。

每当起点和终点确定，一个连续片段便随之产生。

“程序真的把二十八个片段全部列了出来。”托马斯说道。

工程师点点头。“而公式不需要打印任何片段，只要数有多少对边界。”

如果只想知道总数，选择边界更直接。

如果需要查看每一个片段，程序就必须真的移动起点与终点，把它们逐个生成。

数学利用结构压缩工作。

机器则依靠速度完成重复工作。

Notch拿起两张都只写着 `E` 的纸条。

“程序为什么打印了两次相同内容？”

“因为它们来自不同位置。”托马斯回答，“现在的规则按照位置统计，不按照文字内容去重。”

“如果规则改变呢？”

“程序也必须改变。机器不会自己知道哪一种‘相同’才是我们想要的。”

红石工程师在旁边点头。

“机器部分没有问题。”

托马斯看了他一眼。“这次确实还没有。”

当天晚上，托马斯在笔记本上写下：

**一段连续的对象，由开始和结束唯一决定。**

**一排 \(n\) 个对象有 \(n+1\) 条边界。**

**所有非空连续片段共有：**

\[
\binom{n+1}{2}
=
\frac{n(n+1)}2
\]

他又在下面补充：

**连续选择与任意选择不同。**

**连续选择不能跳过中间对象。**

刚写完，财政大臣便送来一份庆典人员名单。

王国即将举办丰收节。参加庆典的居民中，有些报名了矿石展览，有些报名了建筑比赛，还有些报名了红石展示。许多人同时参加两个活动，甚至有人三个活动全部报名。

财政大臣原本把三个报名人数直接相加，结果发现统计出来的“参与者人数”，竟然比王国当天进入广场的人还多。

“人没有变多。”财政大臣说道，“名单上的人却越加越多。”

托马斯看向三张互相重叠的报名表。

铁路上的每段旅程，都有唯一的起点和终点。

可这一次，同一个人可以同时出现在好几张名单里。

如果直接相加，同一个人就可能被数两次，甚至三次。

新的问题已经不再是怎样确定一段连续区域。

而是：

**当几个类别发生重叠时，怎样把重复的人重新数对。**
