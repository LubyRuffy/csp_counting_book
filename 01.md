# 第一课 为什么不能一个一个数？

## 第一部分 仓库管理员的第一天

托马斯第一次站在皇家第一仓库门前时，觉得这份新工作看起来相当体面。

仓库的外墙由整齐的石砖砌成，屋顶铺着深色橡木，门口挂着一块镶金木牌，上面写着“皇家第一仓库”。木牌下方还有一行很小的字：“未经允许不得入内，苦力怕除外。”托马斯盯着最后几个字看了许久，最后决定那大概是前任管理员留下的笑话。毕竟不会有人真的允许苦力怕进入仓库，人类虽然经常做出令人费解的决定，但通常还没有费解到这个程度。

他今年十三岁，昨天还是森林边上的普通少年，日常工作主要是砍树、挖矿，以及在天黑前判断自己能否跑回家。今天一早，皇家信使送来了一套蓝绿色制服、一串沉重得可以当武器使用的钥匙，还有一封只有两行字的任命书：

**任命托马斯为皇家第一仓库见习管理员。**

**负责整理并统计王国物资。**

托马斯觉得第二句话没什么可怕的。整理东西需要耐心，统计东西则只需要会数数。他六岁时就能数到一百，八岁时甚至试图统计森林里一共有多少棵树。那次工作最后没有完成，因为树会继续生长，他父亲又在另一头不断砍伐，整个森林对统计工作表现出一种明显的不配合。

他正在尝试分辨哪一把钥匙属于大门时，身后传来一个声音：“第一天？”

托马斯回过头，看见一个穿着简单黑色短衣的男人。那人留着深色胡子，手里拿着一块普通的橡木原木，正翻来覆去地观察，仿佛这块木头里藏着世界运行的秘密。

“是。”托马斯回答，“我是新来的仓库管理员。”

“见习管理员。”男人纠正道。

托马斯低头看了看任命书。那三个稍小一点的字确实很难忽略。

男人又问：“会数东西吗？”

“当然会。”

“能数到多少？”

“只要时间够，多少都可以。”

男人笑了一下。“那你应该能顺利处理前几只箱子。”

“前几只？”

托马斯还没来得及追问，对方已经把橡木原木放在门边，转身走向王城广场。恰好有一名搬运工推着矿车经过，托马斯便问：“刚才那个人是谁？”

“Notch。”

“他也在仓库工作？”

搬运工摇头。

“那他负责什么？”

搬运工思考了一会儿，最后说道：“大概负责在别人开始怀疑之前，先知道哪里会出问题。”

这是一个并没有让事情变得更清楚的回答。托马斯只好继续研究钥匙。试到第九把时，仓库大门终于伴随着沉重的摩擦声缓缓打开。

门后不是一间仓库，而是一片由箱子组成的森林。

成排的货架从门口一直延伸到大厅深处，高处的箱子几乎顶到屋梁。墙边堆着橡木、白桦木、云杉木、圆石、玻璃、铁锭和颜色各异的羊毛，另一侧还放着矿车、铁轨、红石粉、活塞、胡萝卜、土豆以及一些连标签都写不明白的东西。

托马斯经过一只木箱时，看见上面贴着：

**“暂时不知道有什么用，但以后可能会用到。”**

旁边整整堆了十二箱。

他立刻认出，这些箱子一定属于经验丰富的玩家。真正的新手会丢掉没用的东西，经验丰富的人则会把它们保存十年，以便在第十一年继续确认它们仍然没用。

一只鸡从最高处的货架后面探出头来。

托马斯抬头看着它。

鸡也低头看着托马斯。

没人知道它是怎么上去的。鸡在这个世界里拥有一种特殊能力，总能出现在既不合理、也不需要鸡的地方。

“这只是第一仓库？”托马斯问身边的搬运工。

“对。”

“王国还有几个？”

搬运工指了指他腰间的钥匙。

托马斯数了一下，一共十二把。

他忽然明白，Notch所说的“前几只箱子”并不是夸张。

这时，门外传来整齐的脚步声。国王带着皇家建筑师和两名卫兵走进仓库。国王手里拿着一卷新城墙的图纸，显然不是来欢迎新员工的。国王管理王国的方式一向很直接，因为每省略一句寒暄，就能多提出一个要求。

“北城墙明天开工。”国王说道，“中午以前，我要知道仓库里有多少木材可以用来搭建脚手架。”

托马斯看了一眼四周。“所有木材？”

“原木和木板。”

“木棍算吗？”

“不算。”

“箱子和木门呢？”

“也不算。”

建筑师补充道：“不同树种要分别记录。云杉木要留给塔楼，白桦木用于观景台，其他木材可以用于脚手架主体。”

“为什么观景台一定要使用白桦木？”托马斯问。

建筑师平静地回答：“因为国王觉得它比较明亮。”

这显然不是一道数学问题，甚至可能也不是一道建筑问题。

国王看了看墙上的钟。“你有半个小时。”

说完，他卷起图纸，转身离开了仓库。

附近的搬运工全都停了一瞬，然后若无其事地继续工作。他们显然很想知道，一个昨天还在森林里砍树的人，准备怎样在半小时内数清整个皇家仓库。

托马斯拍了拍手，自言自语道：“不就是数数吗？”

他打开第一只箱子。里面整整齐齐放着二十组橡木原木，每组六十四块。他在账本上写下“橡木原木，二十组”。

第二只箱子里有十八组橡木原木和七组白桦木板。托马斯分别记下两个数字，在旁边画了一个圈，提醒自己不能把它们混在一起。

第三只箱子里装着云杉木板、几组圆石、一把耐久度只剩一点的木铲，以及一块写着“以后再整理”的木牌。托马斯不知道是谁认为木铲应该和建筑材料住在一起，但仓库似乎一直遵循一种古老原则：只要箱盖还能合上，里面的东西就算已经整理完毕。

最初几分钟，一切还算顺利。他打开箱子，数清木材，记录数字，再走向下一只箱子。为了避免忘记哪些箱子已经统计过，他在每只数过的箱子前放了一块圆石作为标记。

这个方法维持了不到三分钟。

一名搬运工推着空矿车经过，看见地上的圆石，顺手全部捡进了车里。

“等等！”托马斯追过去，“那些是我的标记。”

搬运工看了看矿车里的圆石，又指向远处正在修补墙壁的工人。

“哼。”

“我知道圆石可以修墙，可我正在用它们记录哪些箱子已经数过。”

搬运工又看了一眼地面，指向一块写着“保持通道畅通”的牌子。

“哼。”

托马斯大概听懂了。在村民的语言中，这一声“哼”可能同时表达了“你的理由并不能改变它挡路的事实”和“仓库不是你一个人的草稿纸”。

他只好换用粉笔，在数过的箱盖上画叉。

这个办法坚持得更久，直到托马斯发现仓库里的箱子本来就布满各种粉笔标记。有的叉表示“已经检查”，有的表示“箱锁损坏”，有的表示“蜘蛛出没”，还有一只箱子上画着三个大叉，下面写着：

**千万不要打开。**

托马斯把耳朵靠近箱盖，里面传来轻微的爬动声。

他决定尊重前任管理员留下的知识。人类文明能够延续，很大程度上依靠后来者偶尔愿意相信“千万不要打开”并不是一种邀请。

十分钟过去，托马斯开始记不清自己刚才写的是八百九十六，还是九百八十六。两个数字只是交换了一下位置，却足以让城墙少修一大段。

十五分钟过去，一名村民抱着三组橡木原木走进仓库，直接放进托马斯刚刚统计完的箱子。

“这是新送来的？”托马斯问。

村民点头。

“我刚才已经数过这个箱子。”

村民再次点头。

“那我现在应该加三，还是重新数整只箱子？”

村民认真思考了一会儿，发出一声十分深沉的“哼”，随后转身离开。

托马斯逐渐怀疑，“哼”并不是一种语言，而是一项把问题原封不动退还给提问者的技术。

二十分钟过去，又有两组木板被搬去维修城门。账本上的数字刚写完就开始过期。托马斯不得不在旁边不断加减，整张纸很快写满数字、箭头和被划掉的结果，看起来不像库存记录，更像红石工程师在装置爆炸前留下的最后一份线路图。

为了减少混乱，他尝试按箱子统计。第一排箱子合计多少，第二排箱子合计多少，最后把各排加起来。

但同一排里既有原木，也有木板，还有木门、木棍和工作台；同一种橡木又分散在好几排。按箱子分组，只是把一场巨大的混乱切成了几场较小的混乱。

他又尝试按颜色分类。浅色木材放在一边，深色木材放在另一边。这个方法看起来很直观，直到去皮橡木、白桦木板和云杉台阶同时出现在面前。村民坚定地把木棍放进浅色区，因为它确实颜色很浅；另一名搬运工则把深色橡木门放进了云杉木堆，因为它看起来也很深。

托马斯站在两堆材料中间，发现按颜色分类可以让仓库看起来漂亮，却不能回答国王的问题。

二十五分钟过去，仓库深处突然传来一声很轻的声音。

“嘶——”

所有搬运工同时停止动作。

托马斯也停下了笔。

这种声音在方块世界里没有太多解释空间。

一只苦力怕从两排箱子之间走了出来。它神情平静，步伐缓慢，仿佛只是来检查仓库是否符合消防规定。

托马斯举起双手。“别动。”

苦力怕真的停下了。

“我们可以谈谈。”

苦力怕开始膨胀。

“至少不要靠近木材！”

轰的一声，三只箱子飞了起来。原木、木板、台阶、木棍和工作台散落满地。托马斯的账本被气浪卷到半空，在仓库里旋转两圈，最后精准地落进一只装满水的炼药锅。

高处那只鸡也掉了下来。

它扑腾着翅膀，稳稳落在托马斯头顶，仿佛整场爆炸只是它策划已久的下楼方案。

烟尘散去后，已经数过的箱子和没有数过的箱子混在一起，新送来的木材散落在旧库存之间，原来的标记也消失了一半。湿透的账本上只剩三个能够辨认的数字，以及一只清晰的鸡脚印。

托马斯坐在地上，看着满仓库狼藉。

他第一次承认，自己可能不是数得太慢。

而是从一开始，数法就有问题。

## 第二部分 不是不会数，而是不知道该数什么

“半小时到了。”

Notch的声音从门口传来。

托马斯回过头。Notch正靠在仓库门边，手里仍然拿着早晨那块橡木原木。他望着满地散落的材料，没有惊讶，也没有笑话托马斯，似乎苦力怕爆炸只是这个世界表达意见的一种常见方式。

“我会数。”托马斯说道，“可是箱子里的东西混在一起，物资还一直搬进搬出。我不知道哪些已经统计过，也不知道应该把什么加在一起。”

Notch走到一只炸开的箱子旁，先捡起一块橡木原木，又拿起一块白桦木板。

“你刚才在数什么？”

“木材。”

Notch举起橡木原木。“这是木材吗？”

“是。”

他又举起白桦木板。“这个呢？”

“也是。”

Notch指向地上的木棍、木门、工作台和那把耐久度只剩一点的木铲。“它们也由木头制成。”

托马斯张了张嘴。“但国王不要这些。”

“那你数的就不是‘所有木头’。”

Notch把两块材料放回地上。“你连对象都没有说清楚，就开始拼命加数字。爆炸只是让问题提前承认自己存在。”

托马斯低头看着湿透的账本。他原本以为数数只需要认识数字、会做加法，并且足够认真。现在他第一次发现，真正困难的也许不是从一数到多少，而是从一开始就没有说清什么应该算进去，什么应该留在外面。

“那我该怎么办？”他问。

Notch没有回答，只在一块木牌上写下四个字：

**不重，不漏。**

“先别数。”他说，“先把世界切对。”

说完，他把木牌插在地上，转身离开了仓库。

托马斯看着那四个字，开始重新整理问题。

国王要的是能够搭建脚手架的原木和木板。木门、木棍、台阶、箱子、工作台和木铲都不属于统计目标，因此它们应该被明确排除。

不同树种还要分别记录，因为它们的用途不同。

于是，托马斯给仓库划出几个区域：

橡木原木。

白桦木原木。

云杉原木。

橡木木板。

白桦木板。

云杉木板。

不属于脚手架材料的物品则统一进入“其他物品区”。为了防止某位热心村民看到空位后再次塞进几组木板，托马斯给这一区域挂了一块很大的牌子：

**这里的东西不参加本次统计。不要因为看见空箱子就往里塞。**

他还单独设置了“待入库区”和“待出库区”。统计期间新送来的材料先放在那里，不能直接加入已经数过的箱子；准备搬走的材料也先登记，再离开仓库。

搬运工们最初觉得这套规则有些麻烦。毕竟过去的仓库管理原则非常简单：看见空箱子就放进去，箱子满了就找下一只，实在没有空箱子便制作更多箱子。这个方法在短期内极其有效，长期效果则是任何人都不知道东西究竟在哪里。

托马斯没有再按照箱子、房间或颜色分类，而是按照国王真正关心的用途和材料种类来分。

每一块被统计的木材都必须找到唯一的类别。

一块橡木原木只能进入“橡木原木”，不能同时进入“浅色木材”和“建筑材料”两个统计项目，否则相加时会被数两遍。

任何一块符合要求的材料也不能无处可去，否则就会被漏掉。

这正是木牌上“不重，不漏”的意思。

仓库并没有变小，箱子也没有减少，但问题突然变得清楚了。托马斯不再同时面对一座混乱的仓库，而是面对几个能够分别处理的小问题：

橡木原木有多少？

白桦木原木有多少？

云杉木板有多少？

每一类数清之后，再把国王需要的类别合在一起。

这一次，统计没有依赖托马斯的记忆，也不怕某块圆石被搬走。分类本身就在帮助他检查：每一种目标材料都有位置，每个位置的含义都很清楚。

几个小时后，新的报告完成了。

橡木原木二百四十七组。

白桦木原木九十一组。

云杉原木一百八十三组。

其他原木五十六组。

原木总计五百七十七组。

各种木板总计三百七十一组。

因此，可以用于脚手架的木材共有九百四十八组，零散材料另附明细。

国王检查完报告，问道：“确定没有重复？”

“每一件材料只属于一个统计类别。”

“没有遗漏？”

“所有符合脚手架要求的原木和木板，都有对应类别。其他木制品被明确排除。”

建筑师翻到白桦木那一页，开始认真考虑是否应该把观景台缩小一些。国王显然很喜欢明亮的颜色，但白桦树并不会因为国王的审美而加快生长。

托马斯走出议事厅时，Notch正站在城堡台阶上，手里把玩着一枚鸡蛋。那只仓库里的鸡似乎已经开始为王国制造新的库存问题。

“我学会分类了。”托马斯说道。

Notch看了他一眼。“你学会的不是把东西摆整齐。”

“那是什么？”

“你第一次明白，一个问题数不清，不一定是因为数量太大。也可能是因为你还没有把它切成正确的小问题。”

托马斯回头望向皇家仓库。那里面仍然有成百上千只箱子，可一旦所有材料被分成互不重叠、没有遗漏的类别，原本巨大的问题就变成了几个简单的问题。

后来，数学家把这种思想称为**分类加法原理**。

名字听起来很庄严，实际意思却十分朴素：

如果所有对象可以被分成几个互不重叠的类别，那么先分别数清每一类，再把它们相加，就能得到总数。

\[
\text{总数}
=
\text{第一类}
+
\text{第二类}
+
\text{第三类}
+\cdots
\]

真正困难的从来不是最后那个加号。

真正困难的是，在加之前，能不能把类别分得不重也不漏。

## 第三部分 程序员时间：机器不会替人想清楚

第二天，托马斯带着新的统计表来到皇家红石实验室。

红石工程师正在调试一台自动木材统计机。机器由漏斗、箱子、比较器和一排闪烁的红石灯组成，入口处挂着一块写得十分自信的木牌：

**只要是木头，本机都能统计。**

托马斯看到这句话，立刻产生了一种不太好的预感。

工程师把一组橡木原木放进入口。机器发出“咔哒”一声，计数器增加六十四。

他又放入一组白桦木板，机器再次正确增加。

一名村民从旁边经过，顺手把一扇橡木门扔了进去。

机器照样计数。

铁匠放入一把木剑。

机器仍然计数。

托马斯依次放进工作台、栅栏、告示牌和一艘橡木船。计数器每次都忠诚地增加，仿佛王国准备在脚手架上安装门、围栏，并为每名建筑工人配备一艘船。

“它把所有木制品都算进去了。”托马斯说道。

工程师检查了一遍线路。“机器部分没有问题。它的规则就是：只要由木头制成，就进行统计。”

“可国王只要原木和木板。”

工程师停顿了一下。“那就是规则部分有问题。”

他说得十分坦然。机器严格执行错误规则时，红石工程师通常认为它表现得尤其优秀。

托马斯把已经确定好的材料类别交给他。工程师没有继续增加活塞和比较器，而是在黑板上写了一段很短的程序：

```cpp
#include <iostream>
#include <map>
#include <string>
using namespace std;

int main() {
    int n;
    cin >> n;

    map<string, int> count;

    while (n--) {
        string category;
        int amount;
        cin >> category >> amount;
        count[category] += amount;
    }

    for (auto item : count) {
        cout << item.first << ' '
             << item.second << '\n';
    }
}
```

程序每次读入一种已经分好的材料类别和数量，再把相同类别的数量累加起来。橡木原木只进入橡木原木的记录，白桦木板只进入白桦木板的记录；木门和木剑根本不会被送入这次统计。

程序最终输出的结果与托马斯的账本完全一致。

“现在机器学会分类了。”工程师满意地说道。

托马斯摇了摇头。“分类是我们完成的。机器只是按照已经确定的类别加数字。”

计算机不知道什么是“适合搭建脚手架的材料”。它不懂木门为什么不能算，也不知道国王为什么坚持使用白桦木修建观景台。它只会执行人写下的规则，而且执行得非常快。

规则正确，它会迅速给出正确结果。

规则错误，它也会迅速给出错误结果，并且不会产生丝毫怀疑。

程序不会怀疑自己。

苦力怕也不会。

Notch站在实验室门口，问道：“机器最擅长什么？”

托马斯回答：“按照清楚的规则重复工作。”

“最不擅长什么？”

托马斯看了一眼那台曾经把橡木船算进脚手架的机器。

“替我们决定规则。”

Notch轻轻点头。

当天晚上，托马斯翻开新账本，在第一页写下三句话：

**开始计数以前，先说清楚究竟要数什么。**

**分类必须不重复，也不能遗漏。**

**机器可以代替人重复计算，却不能代替人想清楚问题。**

他刚合上账本，一名皇家卫兵便送来了国王的新命令。

军械库正在为北方远征队准备装备。每名骑士必须选择一件武器、一套盔甲和一个背包。武器有三种，盔甲有四种，背包有两种。

国王想知道，一共能组成多少套不同的完整装备。

托马斯看着三个数字，很自然地写下：

\[
3+4+2=9
\]

笔尖刚离开纸面，他忽然停住了。

昨天，分类以后应该相加，是因为一块木材只属于其中一类。

可骑士不会只拿武器，或者只穿盔甲，或者只背一个空背包。他必须完成三次选择，才能得到一套完整装备。

托马斯低头看着纸上的九。

Notch从门外经过，只看了一眼算式，什么也没有解释。

他只问了一句：

“这一次，你数的是三堆东西，还是一条完整的选择过程？”
