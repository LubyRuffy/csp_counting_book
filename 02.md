第二课 为什么有时候要相加，有时候却要相乘？

第一部分 九套装备够不够

第二天清晨，托马斯带着新账本来到皇家军械库。

北方雪原最近出现了一队掠夺者。它们在村庄外竖起旗帜，射坏了两盏路灯，抢走三捆小麦，还试图把铁傀儡引进一条山沟。铁傀儡没有上当，只是站在村口盯着它们。掠夺者被盯了很久，最终决定去找一些眼神没有那么坚定的对手。

国王准备派出一支骑士队，将掠夺者赶回前哨站。远征之前，每名骑士都要领取一套完整装备。

托马斯推开军械库的大门，煤炭、皮革和铁锭混合而成的热气立刻扑了过来。墙上挂着武器，地面立着盔甲架，熔炉里的火仍在燃烧。几名铁匠围着铁砧工作，敲击声此起彼伏，听起来像一群骷髅正在用木剑演奏打击乐。

一名年轻骑士站在角落的镜子前，头上套着一只雕刻南瓜。

托马斯看了他一会儿，问道：“这是远征队的新头盔吗？”

骑士隔着南瓜闷声回答：“不是。”

“那你为什么戴着？”

“昨天有人说它附过魔，让我试一试。”

“什么附魔？”

“绑定诅咒。”

托马斯点了点头，觉得有些问题还是不要继续追问比较好。一个明明写着“绑定诅咒”的东西依然有人愿意试戴，说明王国的装备安全教育还有很大的提升空间。

皇家铁匠从熔炉旁走了过来。他身材高大，围着厚厚的皮围裙，胡子上沾着煤灰，右手握着一把看起来足以让铁砧主动配合工作的锤子。

“你就是新来的仓库管理员？”

“见习管理员。”托马斯说道。

铁匠点点头。“那昨天仓库被炸的事情，是正式管理员负责，还是见习管理员负责？”

托马斯决定假装没有听见。

铁匠把他带到三排装备架前。第一排摆着三种武器：铁剑、弓和弩。第二排放着四套盔甲：皮甲、锁链甲、铁甲和钻石甲。第三排则挂着两种背包：普通补给包和药水包。

普通补给包里装着面包、火把和备用工具。药水包里装着治疗药水、迅捷药水，以及一桶牛奶。牛奶可以解除中毒、缓慢和不祥之兆，也会顺便清除所有有用效果。它对好状态和坏状态一视同仁，公平得近乎不讲道理。

铁匠说道：“每名骑士必须选择一件武器、一套盔甲和一个背包。国王想知道，一共有多少种不同的完整装备。”

托马斯翻开账本。

武器三种。

盔甲四种。

背包两种。

他昨天刚学过分类以后相加，于是很自然地写下：

3+4+2=9

“九种。”托马斯说道。

铁匠没有评价，只从架子上取下一把铁剑、一套皮甲和一个普通补给包，把它们放到桌上。

“这是一套装备。”

随后，他保留铁剑和皮甲，只把普通补给包换成药水包。

“这也是一套。”

托马斯看着桌面。两套装备只差一个背包，但显然应该算作两种不同搭配。

铁匠又把皮甲换成锁链甲。

“第三套。”

再换成铁甲。

“第四套。”

托马斯低头看向纸上的九，隐约感觉这个数字正在失去尊严。

铁匠问道：“一个骑士拿到铁剑以后，还需要什么？”

“盔甲和背包。”

“如果拿到弓呢？”

“也需要盔甲和背包。”

“所以你数的是装备架上的物品种类，还是骑士最终拿走的完整装备？”

托马斯看着三排货架。

他确实数清了货架上有三类东西，却没有数清它们能够组成多少套装备。

第一章中，橡木原木、白桦木板和云杉原木彼此分开。一块材料只属于一个类别，所以应该把各类数量加起来。

但骑士装备不同。

一名骑士不能只选择武器，也不能只选择盔甲。三步必须全部完成，才算得到一套完整装备。

“九不是答案。”托马斯说道。

铁匠点点头。“至少它写得很整齐。”

托马斯把九划掉，开始一套一套列举。

第一套：

铁剑、皮甲、普通补给包。

第二套：

铁剑、皮甲、药水包。

第三套：

铁剑、锁链甲、普通补给包。

第四套：

铁剑、铁甲、药水包。

写到这里，铁匠敲了敲桌面。

“你漏了铁剑、锁链甲和药水包。”

托马斯只好在中间插入一行。

继续写了一会儿以后，他又发现“铁剑、铁甲、药水包”出现了两次。第一次写在第四行，第二次写在第九行，只是三件装备的书写顺序不同。

同一套装备不会因为先写背包还是先写盔甲，就突然变成另一套。可在托马斯的账本里，它已经光荣服役了两次。

他划掉重复的一行，又给每一种装备编号。纸上的数字越来越多，箭头也越来越多，很快便出现了军械库记录常见的状态：写的人觉得自己条理清楚，看的人觉得这里刚刚发生过一次小型战争。

那只曾经从仓库高处掉下来的鸡，不知为什么也来到了军械库。它跳上桌面，从托马斯的清单上慢慢走过，留下几个清晰的泥脚印。随后，它叼起一根羽毛，像完成了某项重要检查一样离开。

托马斯看着被踩脏的账本，第一次对这只鸡产生了一点敬意。它只用了几秒钟，就准确表达了这份清单目前的可靠程度。

“我可以把所有搭配画出来。”托马斯说道。

他在纸上画出三把武器，又在每把武器后面画出四套盔甲，最后从每套盔甲向两种背包连线。

最开始，图还算清楚。画到第二把武器时，线条已经开始互相穿过；画到第三把武器，整张纸看起来像一只洞穴蜘蛛喝下迅捷药水后织出的网。

托马斯放下炭笔。

一个一个列，容易重复，也容易漏掉。

全部画出来，理论上可以，却会很快乱得无法检查。

他需要的不是更大的纸，而是一种能够看清这些选择关系的方法。

这时，Notch提着三只小箱子走进了军械库。

他没有询问托马斯算到哪里，也没有看那张被鸡踩过的图，只把三只箱子依次摆在长桌上。第一只箱子画着一把剑，第二只画着胸甲，第三只画着背包。

Notch在桌子左端放下一只木制小人。

“假设一名骑士从这里走到桌子另一端。”他说，“每经过一只箱子，都必须拿一件物品。”

托马斯看着三只箱子。

“他会做三次选择。”

“离开桌子时，手里有什么？”

“一件武器、一套盔甲和一个背包。”

“所以国王要数的，是三堆分开的东西吗？”

托马斯摇了摇头。

“是一条完整的选择过程。”

Notch没有继续解释。他把小木人留在桌上，转身走向门口。

铁匠望着他的背影，问道：“他为什么总在说到最关键的时候离开？”

托马斯想了想。

“大概是因为剩下的事情应该由我们完成。”

铁匠点点头。“这习惯很适合当顾问。”

第二部分 每一条旧路都会长出新路

托马斯没有再试图一次看清全部搭配。

他先固定武器。

第一种武器是铁剑。

选定铁剑以后，仍然可以从四套盔甲中任选一套。托马斯把皮甲、锁链甲、铁甲和钻石甲依次摆在铁剑后面。

接下来，每套盔甲又都可以搭配两种背包。

铁剑加皮甲，可以配普通补给包，也可以配药水包。

铁剑加锁链甲，同样有两种。

铁甲和钻石甲也一样。

因此，铁剑这一种武器下面，共有：

4\times2=8

套完整装备。

托马斯换成弓。

弓仍然可以搭配四套盔甲，每套盔甲仍然可以搭配两种背包，所以弓下面也有八套。

换成弩以后，还是八套。

于是，所有完整装备可以先按照武器分成三类：

使用铁剑的八套。

使用弓的八套。

使用弩的八套。

这三类之间不会重叠。一套装备不可能既以铁剑为主武器，又同时以弓为主武器，所以可以使用上一章的加法原理：

8+8+8=24

因为每一类的数量都相同，也可以写成：

3\times8=24

再把八展开：

3\times4\times2=24

托马斯看着这三个数字，终于明白它们为什么应该相乘。

第一个三，代表武器的选择。

第二个四，代表选定武器以后，盔甲的选择。

最后一个二，代表武器和盔甲都确定以后，背包的选择。

每一件武器都能接上四套盔甲。

每一种武器和盔甲的组合，又都能接上两个背包。

选择不是彼此分开的，而是一步接着一步。每增加一步，前面已经形成的所有道路都会按照新一步的选择数量再次分叉。

一条路变成两条。

两条变成八条。

最后变成二十四条完整道路。

乘法并不是把几个数字随便挤在一起。它只是把不断分叉的选择过程，压缩成一行算式。

国王来到军械库时，托马斯把二十四种装备清单递给了他。

“为什么不是九？”国王问。

“因为九只是三类物品种数的总和。”托马斯回答，“而一套装备必须从三类中各选一件。选完武器以后还要选盔甲，选完盔甲以后还要选背包，所以每一步的可能都要接在前一步后面。”

国王点点头。

“也就是说，相加和相乘的区别，不在数字本身？”

“不在。”

“那在哪里？”

“在问题的结构。”

托马斯指着三排装备架。

“如果最后的结果只需要从几个互不重叠的类别中选择一种，就把各类数量相加。”

“如果一个完整结果必须连续完成几个步骤，就把每一步的选择数量相乘。”

国王看了一眼铁匠。

“给他一个更难的问题。”

铁匠似乎早有准备。他从架子上取下弓和弩，又指了指钻石甲。

“弓箭手和弩手不能穿钻石甲。”

托马斯问：“为什么？”

“钻石甲太重，不适合在雪原侦察。”

旁边一名骑士小声说道：“主要是钻石甲只有一套，铁匠舍不得借。”

铁匠转头看了他一眼。

那名骑士立刻开始认真检查自己的鞋带。

新的限制意味着，三种武器后面的选择不再完全相同。

铁剑仍然可以搭配四套盔甲，每套盔甲有两种背包：

4\times2=8

弓只能搭配皮甲、锁链甲和铁甲，共三套盔甲，每套仍有两种背包：

3\times2=6

弩也是六套。

因此，合法装备总数变成：

8+6+6=20

托马斯没有直接写：

3\times3\times2

因为三种武器后面的选择并不完全相同。铁剑有四种盔甲可选，弓和弩只有三种。

他先按照武器分类，在每一类内部使用乘法，再把各类结果相加。

这一次，加法和乘法同时出现了。

铁匠问道：“所以，看见三步选择就一定直接相乘吗？”

“不一定。”托马斯说道，“还要检查每一种前面的选择，是否都拥有同样完整的后续选择。如果不同情况的道路结构不同，就先分类，再分别计算。”

Notch站在军械库门口，显然已经听见了他们的讨论。

“现在你知道什么时候加，什么时候乘了吗？”他问。

托马斯没有直接背诵公式。

他指向三类武器。

“加法是把几条互不重叠的道路合起来。”

随后，他又指向武器、盔甲和背包三排装备。

“乘法是让每一条道路继续向下生长。”

Notch轻轻点头。

后来，数学家把这两种最基本的计数方法分别称为：

分类加法原理。

分步乘法原理。

名字听起来像两条需要认真背诵的规定，实际上它们描述的是两种最常见的世界结构。

有些可能彼此分开，最后汇合，所以相加。

有些选择前后相接，一步接着一步，所以相乘。

托马斯在账本上写下：

不要先看数字。

先看选择之间是什么关系。

因为数字不会主动告诉人应该使用加法还是乘法。三个、四个和两个既可以相加得到九，也可以相乘得到二十四。

两个算式都没有计算错误。

真正决定哪个答案正确的，是它们究竟在描述什么。

第三部分 程序员时间：机器把每条路都走一遍

下午，托马斯带着装备清单来到皇家红石实验室。

红石工程师已经制作了一台自动配装机。机器左侧连接武器架，中间是一圈旋转盔甲架，右侧则安装着两个背包发射口。只要拉下拉杆，机器便会为中央的盔甲架装上一套装备，再把装备名称打印在纸条上。

“它能生成所有装备。”工程师说道。

“包括弓配钻石甲吗？”

工程师停顿了一下。

“目前包括。”

拉杆被按下以后，机器先让盔甲架拿起铁剑，穿上皮甲，再背上普通补给包。

第二次，武器和盔甲不变，只换成药水包。

接下来，机器更换盔甲，继续尝试两种背包。

托马斯很快看出，机器正在按照同样的三步过程工作：

先选择武器。

再选择盔甲。

最后选择背包。

红石工程师把这个过程写成一段短程序：

#include <iostream>
#include <string>
using namespace std;
int main() {
    string weapon[3] = {"铁剑", "弓", "弩"};
    string armor[4] = {
        "皮甲", "锁链甲", "铁甲", "钻石甲"
    };
    string bag[2] = {"补给包", "药水包"};
    int total = 0;
    for (int i = 0; i < 3; i++) {
        for (int j = 0; j < 4; j++) {
            if (i > 0 && j == 3) continue;
            for (int k = 0; k < 2; k++) {
                cout << weapon[i] << " + "
                     << armor[j] << " + "
                     << bag[k] << '\n';
                total++;
            }
        }
    }
    cout << "共 " << total << " 套\n";
}

程序输出二十套合法装备，与托马斯的计算完全一致。

托马斯没有研究每一个程序符号，而是观察三层循环表达的含义。第一层尝试三种武器，第二层尝试盔甲，第三层尝试背包。每当程序走到最里面，就意味着三次选择全部完成，一套完整装备随之产生。

遇到弓或弩配钻石甲时，程序会跳过这条不合法的道路。

“程序真的走完了二十条道路。”托马斯说道。

工程师点点头。

“而乘法和分类没有把每套装备全部列出，却直接算出了道路数量。”

“一个方法适合回答有多少种，另一个适合真正列出每一种。”

工程师再次点头。“机器更喜欢第二种，因为它擅长重复。”

这并不奇怪。机器从不会在第十三套装备时感到无聊，也不会因为鸡踩过纸张便忘记自己列到了哪里。它只是按照规则，把每一条允许的道路走完。

Notch站在实验室门外，问道：“机器知道为什么弓不能搭配钻石甲吗？”

“不知道。”托马斯回答，“它只知道遇到这种组合就跳过。”

“它知道为什么应该先选武器，再选盔甲和背包吗？”

“也不知道。顺序是我们设计的。只要不漏掉选择，先选背包也能列出相同的完整装备。”

“那机器知道什么？”

红石工程师认真想了几秒。

“知道怎样非常快速地服从命令。”

这一次，没有人反对。

当天傍晚，建筑师带着一卷新图纸来到实验室。

王国准备扩建皇家仓库。新大厅计划建成六排货架，每排八座，每座货架分成五层，每层放置四只标准箱。

国王想知道，这座新仓库一共能提供多少个箱位。

托马斯看着图纸上的数字：

六排。

每排八座。

每座五层。

每层四格。

他几乎立刻写下：

6\times8\times5\times4

建筑师却没有马上确认。

他展开图纸，指向大厅右侧的一根石柱。石柱挡住了两座货架，其中一处还需要安装红石灯。

“绝大部分位置确实很整齐。”建筑师说道，“但不是每一个地方都完全一样。”

托马斯看着图纸上那几处被打破的格子。

他刚刚明白，整齐重复的选择可以相乘。

下一步，他需要弄清楚的是：

当世界大部分遵守规律，却偏偏留下几个不听话的例外时，应该怎样把它数清楚。
