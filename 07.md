# 第七课 为什么每件物品只有两个选择，方案却会不断翻倍？

## 第一部分 一只背包引发的六十四种争论

北方巡逻队选定后的第三天，矿山传来了一条不太令人安心的消息。

几名矿工在深板岩层下面发现了一条从未记录过的通道。通道两侧长着蓝绿色的幽匿植物，火把插上去以后，附近的方块会发出微弱的脉动。更深处偶尔传来沉重的脚步声，矿工们一致认为，那里要么藏着一座远古城市，要么住着一只体形非常大、而且明显不欢迎访客的生物。

一名矿工声称自己看见了两只发光的角。

另一名矿工则坚持说，那只是第一名矿工逃跑时把两支灵魂火把看反了。

他们没有继续争论，因为无论哪种说法正确，暂时离远一点都显得很有智慧。

国王决定派出一支小型勘察队，先在通道入口建立营地，再确认里面究竟有什么。骑士已经选好，下一步是准备补给。

托马斯来到军械库时，长桌上整齐摆着六件物品：

一把铁剑。

一张床。

一桶水。

十六支火把。

一桶牛奶。

一组面包。

皇家铁匠站在桌子另一边，逐件介绍用途。

铁剑负责处理僵尸、蜘蛛和那些拒绝通过礼貌交谈解决问题的骷髅。床可以让队伍跳过夜晚，但只限主世界。如果有人坚持在下界使用，床也会提供另一种意义上的“快速结束一天”。

水桶可以扑灭火焰、制造水流，还能在从高处掉落时救命。前提是使用者能够在落地以前把水放出来。落地以后再拿出水桶，它通常只能帮助清洗现场。

火把负责照明。每支探险队出发时都认为十六支足够，进入矿洞半小时后又会开始拆走墙上的旧火把，把黑暗留给后来者。这是一种历史悠久的地下资源循环方式。

牛奶可以清除中毒、缓慢和不祥之兆，也会顺便清除力量、夜视和迅捷。它不区分好效果与坏效果，平等得近乎残酷。

面包看起来最普通，却是六件物品里唯一能在队员饿得走不动时，立刻证明自己比钻石剑更有价值的东西。

国王指了指桌面。

“勘察队有不同分工。负责战斗的人可能需要铁剑，绘图员不一定需要；入口接应人员可以带床，却未必需要牛奶。每件物品都可以带，也可以不带。”

托马斯打开账本。“每个背包固定装几件？”

“不固定。”

“三件？”

“不固定。”

“至少一件？”

国王想了想。“从计数上说，空背包也可以算一种方案。至于是否允许它出发，是之后的规则。”

托马斯看着六件物品。

上一章中，他从十二名骑士里固定选择五名，因此可以使用组合数。可这一次，背包里可能有一件、两件、三件，也可能六件全带，甚至什么都不带。

选择数量不再固定。

他决定先按照物品数量分类。

什么都不带，是一种。

只带一件，可以从六件物品中选择一件。

带两件，可以从六件中选择两件。

接下来还有带三件、四件、五件和六件。

这个思路没有问题，可托马斯刚准备计算，铁匠便拿起铁剑、水桶和面包放进背包。

“这一种属于带三件。”

随后，他把铁剑取出来，换成床。

“这也是带三件。”

又把水桶换成牛奶。

“还是带三件。”

托马斯点点头。“所以要数出所有三件组合。”

铁匠又把桌上的火把放进背包。

“现在变成四件。”

接着，他拿走面包。

“又回到三件。”

那只经常出现在托马斯附近的鸡不知从哪里钻了出来，跳上长桌，随后一头扎进敞开的背包，只剩下脑袋露在外面。

铁匠看着它。“现在算几件？”

“鸡不在可选物品里。”

鸡发出一声短促的叫声，显然认为王国的计数制度长期忽视禽类贡献。毕竟它已经参与过仓库爆炸、军械库验收、铁路试运行和骑士名单打印，实际出勤次数比部分官员还多。

托马斯把鸡抱出来，重新检查六件物品。

按数量分类当然可以：

带零件。

带一件。

带两件。

一直到带六件。

可是，这会把一个看起来很简单的问题拆成七个组合问题。物品再多一些，分类会越来越长。

更重要的是，国王并没有要求他分别汇报每种背包里有几件物品，只想知道所有方案一共有多少。

托马斯尝试直接列举。

第一种，什么都不带。

第二种，只带铁剑。

第三种，只带床。

第四种，铁剑和床。

接着加入水桶以后，又出现只带水桶、铁剑和水桶、床和水桶、铁剑床水桶全带……

写到三件物品时还不算太多。加入火把以后，名单开始变长；再加入牛奶，前面出现过的所有方案似乎都要重新来一遍，只是有些带牛奶，有些不带。

托马斯翻看账本，发现自己已经写了两次“铁剑、水桶、火把”。一次在第十三行，另一次在第二十七行，只是物品顺序不同。

他划掉重复项，又发现“床、牛奶、面包”似乎还没有出现。至于是否真的遗漏，他已经不敢确定。

铁匠看了一会儿，说道：“你的名单越来越像仓库第一天的账本。”

托马斯看着密密麻麻的文字。

“至少这次还没有苦力怕。”

军械库门外立刻传来一声：

“嘶——”

所有人同时转头。

一只鹦鹉站在门框上，骄傲地晃了晃脑袋。

铁匠把锤子放回桌面。“我越来越不喜欢会模仿声音的动物。”

托马斯没有继续列清单。他已经知道，问题一旦开始依赖“我记得这一种好像还没写过”，离重复和遗漏通常就不远了。

他需要重新问那个已经帮助过自己很多次的问题：

**一只背包，究竟由什么信息唯一决定？**

长方形由四条边界决定。

巡逻队由被选中的成员决定。

那么，一只远征背包呢？

托马斯依次看向桌上的物品。

铁剑，带还是不带？

床，带还是不带？

水桶呢？

火把、牛奶和面包呢？

每一件物品似乎都只需要回答同一个问题：

**选，还是不选？**

## 第二部分 每增加一件物品，原来的世界就复制一次

托马斯把其他物品暂时移到一边，只在桌上留下铁剑。

如果可选物品只有铁剑，背包有两种方案。

第一种，带铁剑。

第二种，不带铁剑。

铁匠指了指空背包。“什么都不带，也算方案？”

“算。”托马斯说道，“它可能不是有用的远征方案，但确实是六件物品全部不选的结果。”

空背包不是“没有方案”。

它是一种内容为空的方案。

就像仓库里的空箱子仍然是一只箱子，而且通常占据一个非常真实的位置。财政大臣绝不会因为箱子里没有东西，就同意它不占仓库面积。

托马斯把床放回桌面。

现在有两件可选物品。

对于铁剑，仍然是带或不带。

无论铁剑如何选择，床又可以带或不带。

于是，四种方案分别是：

什么都不带。

只带铁剑。

只带床。

铁剑和床都带。

托马斯用木牌画出两次分叉。第一次决定铁剑，第二次决定床。

每一条原有道路，遇到新物品以后都会分成两条：

一条不带新物品。

另一条带上新物品。

他把水桶放回桌面。

原来的四种方案并没有消失。它们各自复制成两份：

一份保持原样，不带水桶。

另一份在原方案上增加水桶。

因此，四种变成八种。

加入火把，八种变成十六种。

加入牛奶，十六种变成三十二种。

最后加入面包，三十二种变成六十四种。

托马斯没有逐个写出全部背包，却已经看清方案数量怎样生长：

\[
1,\ 2,\ 4,\ 8,\ 16,\ 32,\ 64
\]

最开始没有任何物品时，只有一种空方案。

每增加一件物品，原来每个方案都会出现两个版本：带它，或者不带它。

所以，方案数量每次都会翻倍。

六件物品意味着连续做六次“选或不选”的决定：

\[
2\times2\times2\times2\times2\times2
\]

也就是：

\[
2^6=64
\]

如果有 \(n\) 件彼此不同的物品，每件物品都只能选择一次，并且可以自由决定选或不选，那么全部方案共有：

\[
2^n
\]

种。

底数二不是某条突然出现的规定。

它来自每件物品的两个选择。

指数 \(n\) 也不是为了让公式显得厉害。

它表示一共需要完成 \(n\) 次这样的决定。

托马斯把六十四写进报告。国王看了一眼，又问道：“这个答案包括空背包吗？”

“包括。”

“如果规定至少带一件物品呢？”

“从六十四种中去掉全部不选的那一种，剩下：

\[
2^6-1=63
\]

种。”

铁匠指向桌上的床。“如果还规定不能只带床呢？”

托马斯看了看那张床。

“那就再排除这种方案。”

“为什么？”

“因为只带床进入远古城市，除了睡得比较完整，似乎很难完成其他任务。”

一名骑士在旁边提醒道：“而且地下太深，可能根本不是晚上。”

铁匠点点头。“看来床的实用性还需要评估。”

国王没有马上删除任何方案。

列出所有可能，与判断哪些可能符合任务要求，是两件不同的事。

计数先回答一共有多少种选择。

之后再根据照明、食物、重量和安全条件，筛掉不合格方案。

托马斯又用上一章的组合数重新检查这六十四种背包。

什么都不带，等于从六件物品中选择零件：

\[
\binom60=1
\]

恰好带一件：

\[
\binom61=6
\]

恰好带两件：

\[
\binom62=15
\]

恰好带三件：

\[
\binom63=20
\]

恰好带四件、五件和六件，分别是：

\[
\binom64=15,\qquad
\binom65=6,\qquad
\binom66=1
\]

把所有可能的背包大小合在一起：

\[
1+6+15+20+15+6+1=64
\]

也就是：

\[
\binom60+\binom61+\binom62+\cdots+\binom66
=
2^6
\]

两种完全不同的数法，得到了同一个答案。

第一种数法关注每件物品的选择：

选或不选。

第二种数法先按照背包里物品的数量分类，再计算每一类有多少种组合。

它们并不是碰巧都得到六十四。

它们数的是完全相同的一批背包，只是观察角度不同。

一般来说：

\[
\binom n0+\binom n1+\binom n2+\cdots+\binom nn
=
2^n
\]

Notch直到这时才走进军械库。

他没有像以前那样摆木牌、移动边界或提出第一个关键问题。他只是看了看桌上的六件物品，又看了看托马斯画出的分叉。

“这次是谁提醒你的？”他问。

“没人。”

“你怎么找到六十四的？”

“我没有直接数所有背包，而是看每件物品需要做什么决定。每增加一件物品，原来的每种方案都会分成带它和不带它两种。”

Notch指向其中一只背包。“怎样证明两个不同背包不会得到同样的选择记录？”

托马斯依次指向六件物品。

“如果两只背包不同，就至少有一件物品在一只背包里、却不在另一只背包里。那么它们在这件物品上的‘选或不选’一定不同。”

“会不会有某只背包无法写成六次选择？”

“不会。检查六件物品是否在背包里，就能得到完整记录。”

Notch点点头。

每一种背包都对应一串唯一的选择：

铁剑：选或不选。

床：选或不选。

水桶：选或不选。

火把：选或不选。

牛奶：选或不选。

面包：选或不选。

反过来，每一串完整选择也会产生唯一背包。

依然是不重，不漏。

数学家把从一组对象中任意选择若干个得到的结果，称为这组对象的**子集**。

什么都不选，叫作空集。

全部选中，得到包含所有对象的集合。

其余方案则是大小不同的其他子集。

一个包含 \(n\) 个不同对象的集合，共有：

\[
2^n
\]

个子集。

托马斯看着这个公式，忽然觉得它比六十四更惊人。

十件物品有：

\[
2^{10}=1024
\]

种选择。

二十件物品超过一百万种。

三十件物品则超过十亿种。

每件物品明明只有两个极其简单的决定，可这些决定组合起来以后，可能性会迅速膨胀。人类通常在只有六件物品时称它为“选择自由”，等到有三十件物品时，便开始称它为“管理问题”。

## 第三部分 程序员时间：六只拉杆，六十四个背包

红石工程师听说六件物品能够组成六十四种背包，立刻制作了一块配装控制板。

控制板上并排安装着六只拉杆，每只拉杆下面写着一件物品的名称。

拉杆向下，表示不带。

拉杆向上，表示带上。

工程师先把六只拉杆全部压下，按动确认按钮。机器发出一声清脆的“咔哒”，传送带送出一只空背包。

国王看着背包。

“它有什么用？”

工程师回答：“重量非常轻。”

“还有呢？”

“几乎不可能遗失物资。”

国王没有被说服。

接下来，托马斯拨动铁剑对应的拉杆。机器送出只装铁剑的背包。

再打开面包拉杆，背包里便出现铁剑和面包。

关闭铁剑，打开水桶与火把，又得到另一种方案。

每一种拉杆状态，都对应一个唯一背包。

六只拉杆，每只都有上下两种状态，因此总状态数正是：

\[
2^6=64
\]

红石工程师把拉杆状态写成由零和一组成的编号。

零表示关闭，也就是不选。

一表示打开，也就是选中。

例如：

```text
000000
```

表示什么都不带。

```text
100000
```

表示只带铁剑。

```text
101001
```

按照控制板的顺序，表示带铁剑、水桶和面包。

```text
111111
```

表示六件物品全部带上。

工程师又用一段程序枚举控制板的所有状态：

```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    string item[6] = {
        "铁剑", "床", "水桶",
        "火把", "牛奶", "面包"
    };

    for (int plan = 0; plan < 64; plan++) {
        cout << "方案 " << plan + 1 << "：";
        bool empty = true;

        for (int i = 0; i < 6; i++) {
            if (plan & (1 << i)) {
                cout << item[i] << ' ';
                empty = false;
            }
        }

        if (empty) cout << "空背包";
        cout << '\n';
    }
}
```

程序从零到六十三，依次表示六十四种拉杆状态。

托马斯没有急着研究所有位运算符号。他看到的仍然是那六只拉杆：程序只是用一个数字，保存每只拉杆究竟向上还是向下。

每个数字对应一种状态。

每种状态也都有唯一数字。

机器把全部方案逐个列出，数学则不需要真的生成所有背包，便能直接知道一共有六十四种。

程序运行到一半时，那只鸡再次跳上传送带，跟着一只背包进入机器。几秒钟后，出口送出一只装着牛奶、面包和鸡的背包。

红石工程师检查了控制板。

“鸡对应的拉杆没有打开。”

托马斯把鸡抱出来。“因为控制板上根本没有鸡。”

工程师随即在机器旁挂起一块牌子：

**请勿将模型之外的生物放上传送带。**

鸡站在牌子下面啄了两下，显然认为机器应该扩展模型，而不是限制使用者。

Notch看着屏幕上不断打印的方案。

“程序知道为什么一共有六十四种吗？”

“它不知道。”托马斯回答，“它只是从第一种状态一直走到最后一种。”

“那么，谁看见了每加入一件物品，方案就会翻倍？”

“我们。”

机器能够穷尽全部选择。

人则可以通过结构，在不列出每一种的情况下直接知道总数。

两者都能得到答案，却使用了不同的力量。

机器依靠重复。

数学依靠理解。

当天傍晚，国王批准了几套实际远征方案，并将空背包、只带床和完全不带火把与食物的方案划入“不建议出发”名单。

托马斯刚准备收起账本，铁路售票员便匆匆赶来。

王城到北方矿山之间已经建成九座车站。铁路目前只能从编号较小的车站开往编号较大的车站，售票处需要提前制作所有不同的单程车票。

“每张票需要写一座出发站和一座到达站。”售票员说道，“中间经过的车站不用写，因为铁路只有一条。”

托马斯看向墙上的线路图。

这一次，不是从九座车站中任意挑选若干座。

旅客不能从王城出发，跳过中间铁路，直接把雪原和矿山一起装进背包。

一段旅程必须是连续的。

只要出发站和到达站确定，中间经过的整段铁路便随之确定。

托马斯合上远征背包的记录，重新拿起炭笔。

任意选择需要逐个回答每个对象“选不选”。

而连续的一段，似乎只需要知道：

**从哪里开始，又在哪里结束。**
